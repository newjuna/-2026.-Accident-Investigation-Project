# 사고 발생 현장 대응 가이드

안전보건팀 실무자용 모바일 웹 도구입니다. GitHub Pages로 배포해 URL 하나로 모바일에서 바로 접속할 수 있습니다.
GitHub Pages 특성상 저장소가 공개되면 누구나 URL로 접속할 수 있어, 비밀번호 4자리로 최소한의 접근 제한을 둡니다.

## GPT V1 변경사항

- 사고조사 탭의 `전송하기`를 누르면 버튼/상태 문구가 먼저 화면에 표시된 뒤 전송이 시작되도록 개선했습니다.
- 결과문서 PDF는 휴대폰이 아니라 Apps Script 서버에서 생성해 Drive에 저장하고 Teams 카드에 링크로 첨부합니다.
- PDF Blob이 실제로 만들어진 경우에만 `PDF 공유하기` 버튼을 사용할 수 있게 했습니다.
- 모바일에서 결과문서 캡처가 실패하면 깨진 PDF를 만들지 않고 재시도 안내를 표시하도록 보강했습니다.
- Google Apps Script 응답을 브라우저가 읽지 못하는 경우에도 전송 요청 완료 상태로 안내하도록 조정했습니다.

## GPT V1.1 보정사항

- 한글이 깨질 수 있는 텍스트 대체 PDF 생성을 제거했습니다.
- PDF를 길쭉한 단일 페이지가 아니라 표준 A4 여러 페이지로 생성해 카톡/Teams에서 열었을 때 형태가 무너지는 문제를 줄였습니다.
- PDF 이미지 품질과 용량을 조정해 휴대폰 전송 부담을 낮췄습니다.
- Apps Script 전송 대기 시간을 20초에서 60초로 늘렸습니다.
- 모바일 Teams 앱은 공유시트에서 파일 첨부를 무시하고 메시지만 보내는 경우가 있어, 이 경우 다운로드된 PDF를 Teams 앱에서 파일 첨부로 직접 선택해야 합니다.

## GPT V1.2 보정사항

- 휴대폰에서 `html2canvas`가 결과문서 캡처에 실패해도 캔버스에 문서 내용을 직접 그려 PDF를 만들도록 보강했습니다.
- 이 백업 PDF는 한글을 PDF 텍스트로 넣지 않고 이미지로 굽기 때문에 한글 폰트 깨짐을 줄입니다.
- DOM 캡처 실패 때문에 `PDF를 만들지 못해 전송하지 않았습니다`가 뜨던 흐름을 백업 PDF 생성으로 우회합니다.

## GPT V1.3 보정사항

- 휴대폰 PDF 생성 실패가 계속 발생해도 전송을 막지 않도록 구조를 변경했습니다.
- 프론트는 사고조사 데이터만 전송하고, `apps-script/Code.gs`가 서버에서 결과문서 PDF를 생성합니다.
- 서버 생성 PDF는 Google Drive에 저장되며 Teams 카드의 `결과문서 보기` 버튼으로 연결됩니다.
- 휴대폰의 `PDF 공유하기`는 로컬 PDF가 만들어진 경우에만 사용할 수 있고, 기본 운영 흐름은 서버 생성 PDF 링크입니다.

## 화면 구성

- 로그인 (비밀번호 4자리) → 초기 로딩("시스템 최적화 중") → 본 화면
- 상단 우측 헤더의 햄버거 버튼을 누르면 우측에서 사이드바 메뉴가 열립니다.
- 메뉴: **즉시대응 / 매뉴얼 / 사고조사 / 경위서 작성 / 휴업·산재**
- 헤더의 🔍 버튼을 누르면 즉시대응·사고조사·경위서·휴업신청·FAQ를 통합 검색하는 화면이 열립니다.

## 파일 구성

- `index.html` — 전체 화면 마크업
- `assets/css/style.css` — 스타일 (아성다이소 브랜드 컬러: 네이비 `#0D1B36` + 레드 `#E60012`)
- `assets/js/`
  - `auth.js` — 로그인(비밀번호 4자리) + 초기 로딩 화면 처리
  - `manual.js` — 매뉴얼 탭 (챕터 전환, 통합 검색, 아코디언, 서브탭, FAQ, 이미지 라이트박스)
  - `investigation.js` — 사고조사 탭 (확인자료 체크 + 목격자 면담 + 판단 + 결과 문서 + PDF/Teams)
  - `org-data.js` — 조직도(부문/부서/팀) 3단 드롭다운 데이터
  - `search-data.js` — 통합 검색 데이터 (즉시대응/사고조사/경위서/휴업신청)
  - `faq-data.js` — 매뉴얼 FAQ 28문항 데이터
  - `main.js` — 사이드바 메뉴 전환 + 헤더 검색 화면 전환 + 통합 검색 로직
- `assets/img/` — 매뉴얼·경위서 작성 가이드 이미지

## 로그인 / 접근 제한

- 비밀번호는 `assets/js/auth.js`의 `AUTH_PASSCODE` 상수에서 관리합니다. (임시값: `0000`)
- 로그인 유지 시간은 `AUTH_SESSION_HOURS` 상수로 관리하며 기본 3시간입니다. 이 시간이 지나면
  다시 비밀번호를 입력해야 합니다. (기기 로컬 저장소에만 남는 정보이며 서버로 전송되지 않습니다)
- 비밀번호를 바꾸려면 `auth.js` 상단의 `AUTH_PASSCODE` 값만 수정하면 됩니다.

## 로컬에서 확인하기

별도 서버 없이 `index.html`을 브라우저로 열어도 되고, 정적 서버로 띄우고 싶다면:

```
npx serve .
```

## GitHub Pages 배포

1. 이 폴더를 GitHub 리포지토리 루트로 푸시
2. 리포지토리 Settings → Pages → Branch: `main` (또는 `master`), 폴더: `/ (root)` 선택
3. 몇 분 후 `https://<계정명>.github.io/<repo이름>/` 주소로 접속 가능

## Teams 연동 (선택)

`apps-script/Code.gs`와 `apps-script/README.md`를 참고해 Google Apps Script 중계 서버를 배포하면,
"사고조사" 탭에서 결과를 Teams 채널로 전송하고 Google Sheets에 기록을 남길 수 있습니다.
